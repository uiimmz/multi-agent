请实现一个基于 LangGraph 的多Agent架构的电商商品多模态智能解析系统。系统架构与要求如下：

用户输入包括一张图像（通过文件路径传递）和一段文本。系统由一个主 Agent 和三个子 Agent 组成。所有模型均通过 API 调用，禁止本地模型。

主 Agent 使用 DeepSeek-V4-Pro 模型，通过 OpenAI 兼容接口调用，基础 URL 为官方地址。主 Agent 负责任务规划、路由决策、结果聚合和重试控制。

三个子 Agent 严格隔离，互不调用：
所有apikey在同目录apikey文档下

视觉理解 Agent：调用 qwen3-vl-plus 模型，api接口调用，根据图像和文本输出。

文案生成 Agent：调用 qwen3.6-plus 模型，api接口调用，基于视觉描述和用户指令生成创意文案及置信度。

内容校验 Agent：调用 MiMo-V2.5-Pro 模型，通过 API 接口 接收图像 base64、视觉描述、生成文案，返回校验得分（0~1）校验评分。若未配置真实 API 端点，则使用#请填入api代替。自动启用模拟校验（返回随机0.6~0.9的分数，以及简单评语）

工作流程为：用户输入 → 主 Agent 规划 → 依次调用三个子 Agent → 主 Agent 聚合结果，根据综合置信度和阈值决定直接输出或触发重试，阈值设为0.7。如果解析不到置信度，可以调用主Agent（或规则）根据描述内容估算一个默认值（如0.7），并记录日志。

环境依赖包括 langgraph、openai、dashscope、Pillow、requests 等库，所有密钥通过环境变量读取，不可硬编码。

状态定义需使用 TypedDict，包含以下字段：图像路径、图像 base64、用户查询、任务类型（问答或描述）、执行计划（JSON 结构）、路由决策、视觉描述及其置信度和原始响应、生成文案及其置信度和原始响应、校验结果（评语、分数、是否通过）及其原始响应、最终输出及其置信度、各 Agent 重试次数（最大三次）字典、阈值配置字典、消融配置字典、消融标签列表、日志列表。阈值配置需包含最终阈值、最大重试次数以及各 Agent 的独立阈值。消融配置需包含是否启用视觉、生成、校验三个 Agent 的开关，以及置信度计算模式（模型提供、最小值、加权平均、仅校验）和相应的权重。重试时只重试失败的那个Agent，在重试决策时，跳过被消融的Agent.

每个子 Agent 实现为一个独立的函数，接收状态并返回部分更新。视觉 Agent 从图像路径读取并转换为 base64，构造符合 OpenAI Vision API 格式的消息，要求模型输出结构化描述并在末尾以指定格式给出置信度，调用 DeepSeek-VL 模型，可请求 logprobs，从响应中提取描述和置信度，解析失败时重新解析。文案生成 Agent 类似检查开关，若启用则基于视觉描述和查询构造提示词，要求输出文案和置信度，调用 Qwen-Flash，解析输出，失败时使用默认置信度。校验 Agent 若启用则优先调用真实 MiMo API，返回分数，评语，是否通过。

主 Agent 包含规划节点、路由决策节点和聚合节点。规划节点调用 DeepSeek-V4-Pro 分析用户输入，返回 JSON 格式的计划，包含任务类型和执行步骤列表，解析失败时使用默认计划。路由决策节点根据当前状态中依次调用子 Agent 或进入聚合节点。聚合节点收集所有子 Agent 输出，根据消融配置中的置信度模式计算最终置信度，若低于最终阈值且重试次数未达上限，则找出置信度最低的环节，决定重试对应的 Agent，并在重试前清空该 Agent 的输出字段；否则输出最终结果并结束。

图构建使用 LangGraph 的 StateGraph，添加规划节点、三个子 Agent 节点、聚合节点。设置入口点为规划节点。规划节点后连接路由节点（可作为条件边函数），路由节点根据状态分发到不同的子 Agent 或聚合节点。每个子 Agent 执行后无条件返回路由节点。聚合节点后添加条件边，根据重试决策函数返回要重试的子 Agent 节点或结束。保证子 Agent 之间没有直接相连的边。

置信度计算模式需实现：按配置权重加权平均。所有阈值参数集中在阈值配置字典中，便于实验调整。

消融实验支持通过修改消融配置字典来独立关闭任意子 Agent（除视觉agent和主agent之外），在关闭时该 Agent 填充默认值并记录消融标签。聚合节点需根据开关组合调整最终输出（例如无校验时，最终置信度取视觉和生成的最小值）。提供一组预定义的消融配置（如完整流水线、无生成、无校验等）。

日志功能要求每个节点追加带时间戳的关键步骤到日志列表。提供运行单次流水线的函数，接收图像路径、查询和配置覆盖参数，返回最终状态。提供批量运行函数，接受图像-查询对列表，支持异常容错，并将所有结果保存为 JSON Lines 文件。
